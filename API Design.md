# API Design

Good API design makes an API easy to understand, maintain, and scale.

## Principles

- Use nouns for resources
- Use the correct HTTP methods
- Return meaningful status codes
- Keep responses consistent
- Version your API
- Support pagination
- Validate input
- Write clear documentation

### Interview answer

> A well-designed API should use clear and consistent resource names, follow REST principles, use the correct HTTP methods, return meaningful status codes, support pagination for large datasets, include versioning for backward compatibility, and provide predictable and consistent responses.
